Multi-tool icon button row — mockup

Replaces the single Connect Desktop button with one icon-button per enabled provider. "Run Script" → "Scripts" and "Remote Tools" → "Tools" to make horizontal room.

Before (current)

Single launcher → opens whichever provider is the default (set in Partner Settings → Remote).
GTech-BD-Win11 Online
Windows 11 Enterprise · Agent v0.65.7 · China

After (proposed)

Icon row of every enabled provider. Click an icon to launch that tool against the device. The current default provider (from Partner Settings → Remote) carries an indicator dot and a primary outline so it stays visually distinguishable. Hover for the provider name.

Windows 11 Enterprise · Agent v0.65.7 · China

● The blue dot + outline indicates the current default provider (the one a partner-admin selected in Partner Settings → Remote). Clicking the default icon does the same thing the existing Connect Desktop button does today.

Disabled providers (per the partner setting enabled: false) don't appear in the row at all. A device that's missing the required custom field for a given provider gets a dimmed icon with a tooltip explaining why (e.g. "sc_session_id not set on this device").

On screens narrower than the actions row can fit, the providers collapse into a single icon-button with a popover (icon = current default; arrow on the right opens the picker).

Notes for the v2 implementation

  • This view does not change the data model — it consumes the same partners.settings.remoteAccessProviders array we just shipped, just renders one icon per enabled: true provider.
  • The "Connect Desktop" rendering is a fallback when the partner has no providers configured (or only the built-in is enabled): show a single icon for the built-in WebRTC option, identical to today's behavior.
  • Provider icons are loaded from a partner-uploadable asset slot (or from a built-in catalog with the names rustdesk, screenconnect, teamviewer, anydesk, splashtop). MSPs with a custom-branded build (e.g. our BDunn-RustRemote) can upload their own image.
  • The default-provider indicator stays in sync with whatever defaultProviderId is currently set; toggling default in the settings page reflows the row instantly.
  • Renames are pure label changes — Run Script → Scripts and Remote Tools → Tools. Both icons unchanged.
